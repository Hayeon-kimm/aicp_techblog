## 오늘의 목표

n8n 사용해서 개인용 봇 만들기

## 다음 목표

n8n으로 개인 봇에 기능 추가하기

## 요약

n8n 자동화 플랫폼을 활용하여 디스코드 봇이 테크블로그 봇에게 자동으로 블로그 포스팅을 요청하는 시스템을 구축했어요. 개발 과정에서 n8n 최신 버전 설치를 위한 서버 환경 업데이트 후 디스코드 봇이 응답하지 않는 이슈가 발생했고, 프로세스 재시작으로 해결했어요.

## 환경

- n8n (자동화 워크플로우 플랫폼)
- Discord Bot (for휘준)
- TechBlog_Bot (Claude 기반 블로그 자동화 봇)
- 서버 환경 (Node.js 등)

## 오늘 한 일

### 1단계: 기본 워크플로우 테스트

먼저 n8n에서 간단한 트리거 → 디스코드 메시지 전송 워크플로우를 구성하여 기본 동작을 테스트했어요.

![n8n 기본 워크플로우 - Execute workflow 트리거로 Discord에 메시지 전송](/assets/img/2026-04-08-n8n-discord-bot/n8n-basic-workflow.png)

워크플로우 실행 버튼을 클릭하면 디스코드 채널에 메시지가 자동으로 전송되는 것을 확인했어요.

![디스코드에서 봇 메시지 수신 확인](/assets/img/2026-04-08-n8n-discord-bot/discord-bot-test.png)

### 2단계: 전체 자동 포스팅 워크플로우 설계

기본 테스트 성공 후, 최종 목표인 전체 자동 포스팅 파이프라인을 설계하고 구현을 시도했어요.

![n8n 전체 워크플로우 - Webhook → Get messages → Code → HTTP Request → Send message](/assets/img/2026-04-08-n8n-discord-bot/n8n-full-workflow.png)

**전체 로직 흐름:**

1. 디스코드 봇에게 태그 + "오늘의 포스팅" 문구 전달
2. 봇이 서버에 HTTP 포스팅 → Webhook 트리거
3. 디스코드 봇과 나의 대화 내용 GET (247개 메시지 수집)
4. JavaScript 코드 노드에서 오늘 날짜 스탬프 기준 대화 내용 수렴
5. HTTP Request로 Claude에게 대화 내용 요약/포맷팅 요청 (localhost:3000)
6. 포맷팅된 요약본을 테크블로그 봇에게 디스코드 메시지로 전달

## 에러/이슈

n8n 최신 버전으로 설치하기 위해 서버 환경을 업데이트했더니, 디스코드 봇이 응답이 없어지는 상황이 발생했어요.

### 원인 분석

서버 환경(Node.js 버전, 패키지 의존성 등)이 변경되면 기존에 실행 중이던 프로세스가 변경된 환경을 인식하지 못해요. 이로 인해:

1. 기존 프로세스가 이전 환경의 라이브러리/설정을 참조
2. 새 환경과의 호환성 문제로 정상 동작 불가
3. 디스코드 봇의 WebSocket 연결이 끊어지거나 이벤트 핸들러가 비활성화

## 해결 방법

**환경이 바뀌면 프로세스를 재시작해요.**

```bash
# n8n 프로세스 재시작
pm2 restart n8n
# 또는
systemctl restart n8n

# Discord 봇 프로세스 재시작
pm2 restart discord-bot
# 또는 수동 실행
node bot.js
```

### 예방을 위한 팁

1. **환경 업데이트 전 체크리스트 작성**: 어떤 프로세스가 실행 중인지 확인
2. **프로세스 매니저 활용**: pm2 등으로 관리하면 재시작이 간편
3. **업데이트 후 반드시 전체 서비스 재시작**: 환경 변경 후에는 모든 관련 프로세스를 재시작하는 것을 습관화

## 참고

- 서버 환경 변경 시에는 항상 의존성 프로세스의 재시작을 고려해야 해요.
- n8n과 같은 자동화 도구를 사용할 때는 워크플로우의 안정성을 위해 환경 변경에 주의가 필요해요.
