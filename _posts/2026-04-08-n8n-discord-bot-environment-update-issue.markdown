## 요약

n8n을 최신 버전으로 설치하기 위해 서버 환경을 업데이트했더니, 기존에 동작하던 디스코드 봇이 응답을 하지 않는 문제가 발생했습니다. 환경이 변경되면 관련 프로세스를 재시작해야 정상 동작합니다.

## 환경

- n8n (자동화 워크플로우 플랫폼)
- Discord Bot
- 서버 환경 (Node.js 등)

## 오늘 한 일

n8n을 이용해서 디스코드 봇이 디스코드 봇에게 블로그 포스팅을 하도록 자동화 워크플로우를 구성했습니다.

## 에러/이슈

n8n 최신 버전으로 설치하겠다고 서버 환경을 업데이트했더니, 디스코드 봇이 응답이 없어지는 상황이 발생했습니다.

### 원인 분석

서버 환경(Node.js 버전, 패키지 의존성 등)이 변경되면 기존에 실행 중이던 프로세스가 변경된 환경을 인식하지 못합니다. 이로 인해:

1. 기존 프로세스가 이전 환경의 라이브러리/설정을 참조
2. 새 환경과의 호환성 문제로 정상 동작 불가
3. 디스코드 봇의 WebSocket 연결이 끊어지거나 이벤트 핸들러가 비활성화

## 해결 방법

**환경이 바뀌면 프로세스를 재시작합니다.**

```bash
# n8n 프로세스 재시작
pm2 restart n8n
# 또는
systemctl restart n8n

# Discord 봇 프로세스 재시작
pm2 restart discord-bot
# 또는 수동 실행
node bot.js
```

### 예방을 위한 팁

1. **환경 업데이트 전 체크리스트 작성**: 어떤 프로세스가 실행 중인지 확인
2. **프로세스 매니저 활용**: pm2 등으로 관리하면 재시작이 간편
3. **업데이트 후 반드시 전체 서비스 재시작**: 환경 변경 후에는 모든 관련 프로세스를 재시작하는 것을 습관화

## 참고

- 서버 환경 변경 시에는 항상 의존성 프로세스의 재시작을 고려해야 합니다.
- n8n과 같은 자동화 도구를 사용할 때는 워크플로우의 안정성을 위해 환경 변경에 주의가 필요합니다.
